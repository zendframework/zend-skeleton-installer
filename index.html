zend-skeleton-installer

Composer plugin for providing and prompting for optional components in skeleton projects.

$ composer require zendframework/zend-skeleton-installer

Optional Packages

To define an optional package, you will add an object under a `extra.zend-skeleton-installer` array in your `composer.json`. The object **must** contain the following properties:

  • name: the name of the package to install
  • constraint: the package constraint to use
  • prompt: a textual prompt (e.g., "Do you want to install console tooling?"

You may also define two boolean flags:

  • module: when true, indicates that the component represents a Zend Framework module. This flag will not dictate whether or not the component installer prompts to install the package as a module, but will affect hints provided by Composer during installation.
  • dev: when true, indicates the package should be installed as a development requirement (require-dev). Additionally, the installer will prompt you to choose a development configuration file in which to install the component if it is a module.

Example


{
  "name": "zendframework/zend-skeleton-application",
  "type": "project",
  "require": {
    "php": "^5.6 || ^7.0",
    "zendframework/zend-component-installer": "^0.1",
    "zendframework/zend-skeleton-installer": "^1.0.0-dev@dev",
    "zendframework/zend-mvc": "^3.0.0-dev@dev"
  }
  "extra": {
    "zend-skeleton-installer": [
      {
        "name": "zendframework/zend-mvc-console",
        "constraint": "^1.1.2",
        "prompt": "Would you like to install MVC console tooling?",
        "module": true
      },
      {
        "name": "zendframework/zend-mvc-i18n",
        "constraint": "^0.1",
        "prompt": "Would you like to install i18n support?",
        "module": true
      },
      {
        "name": "zendframework/zend-test",
        "constraint": "^2.6.1",
        "prompt": "Would you like to install MVC testing support?",
        "dev": true
      }
    ]
  }
}

The above will:

  • Prompt if you want to do a minimal install; if so, the plugin will remove the extra.zend-skeleton-installer section on completion, and then delete itself from the installation.
  • Prompt for each of the listed components, as noted, awaiting a y/N answer (with "n" as the default).
  • Install the selections, updating the composer.json.
  • Remove itself from the installation on completion.
